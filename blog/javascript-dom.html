How does JavaScript relate to HTML & CSS

HTML is use to create and provide the basic structure and content of a website. CSS is then used to alter and control the layout, style/looks and formatting of the website
and then JavaScript is used to control how the elements behave when interacted with. JavaScript manages to do this when we use it to create a set of instructions (scripts) that
tell our browser or whatever we are using to view our website how to view the different elements that are on the page. Javascript can allow the website to behave differently when
the user interacts with it for example you can make elements(the content on the page)display different behaviour depending on if that user is hovering or clicking on an element.
An analogy for how HTML CSS and JavaScript interact with each other could be that the HTML is the framing and gib board that make up the structure of your house, CSS is the paint
and decor that you use to make it look appealing. And then finally JavaScript is like a person living in it that can re-arrange furniture turn on lights paint walls different colours
in response to how his mind(the web developer)sees fit at the time.



Control Flow and Loops


A control flow is the order in which JavaScript reads the code that is within the .js file. By default it will always read the instructions(scripts) provided to it from top to
bottom. This is unless JavaScript runs in to a special instruction provided to it that tells it to jump to a different part in the instructions either elsewhere on the page or
to repeat the same script if it is a part of a loop. Or if it is being asked to receive an instruction from a different file. After JavaScript jumps to its newly instructed spot
it will continue to read top to bottom from that new spot unless it is instructed to return to somewhere else or told top stop.



The instructions that change the default control flow (top to bottom) are most commonly loops (repeat a section of code a specified amount of times); function calls (made to purpose
chunks of code that are utilized by other blocks of code) and conditional statements. (perform different actions based on logic created by the web developer)

The difference betwen accessing data from objects and arrays

Objects are used to represent something in your code that can be defined and seperated by a set of characteristics.
In the case of objects the characteristics are referred to as properties that are made up by a key and a value. For example

//Example person
let person ={
name: 'Ash',
age: '22',
learningCoding: True
};
In this example the keys are name,age and learningCoding. The values are Ash,22 and True.
Items in an object can be accessed by either dot notation or bracket notation, allowing items to be added or removed from the objects
For example we can change the value of learningCoding to false by using person.learningCoding = false.


Arrays are used to store a list of multiple items within a single variable. They are even more useful for when you want to create ordered collections that can be accessed
by their numbered place in the list. Items can be added to arrays using push(),pop(),unshift() and shift() depending on where in the array you want to place the item.

  • push() Adds item(s) to the end of an array.
  • pop() Removes the last item from an array.
  • unshift()Adds item(s)to the beggining of an array.
  • shift() removes the first item from an array..

Why are functions useful?

Functions are a set of instructions within the code that perform a task when called from. These are useful as they allow us to create a repeatable task that we can re-use
throughout the code without having to type the same chunk of code out repeatedly to achieve the same result.
